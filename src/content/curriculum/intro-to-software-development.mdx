## What is software development?

Software development is the process of turning an idea into a working digital product. It includes planning, writing code, testing behavior, and releasing updates over time.

At a high level, developers solve real problems by building systems that are clear, reliable, and maintainable.

## Core lifecycle

| Stage | What happens |
| --- | --- |
| Discover | Define the problem, users, and goals |
| Design | Plan flows, data models, and architecture |
| Build | Implement features in code |
| Test | Verify correctness, performance, and usability |
| Deploy | Release to users and monitor behavior |
| Iterate | Improve based on feedback and metrics |

## Foundational skills

1. **Programming basics**: variables, control flow, functions, and data structures.
2. **Version control**: using Git to track and collaborate on changes.
3. **Debugging**: finding root causes instead of patching symptoms.
4. **Testing mindset**: confirming software works before and after changes.
5. **Communication**: writing docs and sharing decisions clearly.

<Alert tone="info" title="Start small">
Build short, complete projects early. A tiny app you can finish teaches more than a large app you never ship.
</Alert>

## Common tools in modern workflows

- **Editor/IDE**: where code is written (for example, Neovim or VS Code).
- **Runtime**: executes code locally (Node.js, Python, etc.).
- **Package manager**: installs dependencies (`npm`, `pnpm`, `pip`).
- **Framework**: gives structure for apps (Astro, Next.js, Django, Rails).
- **CI/CD**: automates tests and deployments.

## First-project roadmap

### Week 1

- Learn basic syntax in one language.
- Build a tiny command-line program.

### Week 2

- Learn Git basics (`init`, `add`, `commit`, `branch`).
- Build a small web page with simple interactivity.

### Week 3

- Add tests to one feature.
- Refactor your code into clearer modules.

### Week 4

- Deploy a small app.
- Write a short readme explaining setup and usage.

By the end, you should have one complete project, one deployment, and one documented learning loop.
